Hosting Our Website With S3

Intro

To start off, we will need to have an Amazon Web Services (AWS) account. AWS is a cloud service provider that we will be utilizing infrastructure from to create our application.

Because we have a "static" website, or a wesbite that doesn't rely on server side coding, we can host the site through AWS's S3, or Simple Storage Service.

Imagine if your application could scale to a million users, and all you had to do was click a button to allow automatic scaling. This is what serverless offers. You don't have to concern yourself with configuring auto-scaling groups and worrying if your settings are working for a real life scenario. Gone...are the days of waking up at 2 a.m. to an enraged client telling you their site is down.

Creating Our Script to Push Changes to Environments

This will also be the first step in creating in our pipeline to push code from out localhost environment to staging or production. We want to create a BASH script so that we can repeatably run a command that we will be using a lot in an easy way.

Create sync-to-s3-live-site.sh File

Make sure you're in your terminal, and then create a file with the .sh extension to begin coding for the BASH language. I like to use the Nano text editor. Once you're in your file, copy and paste the following code:
#!/bin/bash/ aws s3 sync $PWD s3://YOUR-BUCKET-NAME/ --acl public-read

Our Code Explained

Even for those unfamiliar with BASH or the AWS command line, the code in our script should be pretty straight-forward to understand. The first line declares the language to be used for this script. The second line is the actual code, which can also be copy/pasted into your terminal window. It uses the Amazon Web Services command to tell the Simple Storage Service that we want to copy our code from the local machine to the S3 bucket that we created earlier. We set it with public-read permissions so that anyone visiting our website can have access to view it.



Testing Our Script Out

We can test this script out by adding the execute permission to the file. In your terminal, type:
sudo chmod +x sync-to-s3-live-site.sh

After you've run the command above and the file has the correct permissions, you can then type the following to execute the file:
./sync-to-s3-live-site.sh

If you've set your information correctly in the script and are in the same directory as your web application, then you should have successfully uploaded your site to S3. The output will look similar to the screenshot below.
